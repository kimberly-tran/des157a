Project Summary

I plan to update Studio Project 3: Game On! by creating a cleaner design, incorporating more visual and aural feedback, adding more options for player customization.

Comparison Project 1

Looking at this Minesweeper game, the concept and interactions needed in the game itself are very minimal. The UI is also very simple, but the small animations in each click and interaction offers the user a lot of feedback and an overall enjoyable experience. For example, every time the player clicks on a block to check for a “mine,” the block flies away. Additionally, on the button clicks there is a bounce animation as well as a clicking noise to indicate the users’ action, which was also a good touch to give the user feedback for their actions. As for other audio components, there are “positive” and “negative” sound feedback for when the player clicks on a “safe” block or a block with a mine in it. The audio track played throughout the game does not start until a player starts the game, and any time the player leaves the page, the music pauses which I thought was a nice feature, since when people switch tabs they are usually now switching their focus to something new. Another successful feature was the introduction of game rules (”tutorial”) at the beginning and then having them nested in the pause/settings later on instead.

Comparison Project 2

In this game, Squish Run, both the mouse and keyboard are utilized — the mouse primarily for clicking and the keyboard for in-game movement. The graphics itself are very appealing, as they are bright pixel characters/illustrations and incorporate subtle animations to give a sense of movement and reality. Something that stood out to me that was working well was the option that users had of being able to switch between and unlock new character skins to play with. I think this offers another fun component to not only keep users still playing, but let them feel like they’re making the game their own. Overall, the playing experience uses minimal button keys and on-screen animations, but the variety in maps, use of levels and character upgrades, keeps players interested.

Project Goals

For this project, I want to incorporate more subtle animations and audio components that will serve to provide the user with more effective feedback, as well as keep players more interested in the game as a whole. Similar to the minesweeper game, I plan to incorporate animations and audio on button clicks, and utilize different audio tracks to give the user either “positive” or “negative” feedback for when they win or lose (i.e. roll snake eyes). I also plan to include more options for the user to hide/view game instructions, control audio tracks, etc. Another goal I have for this project is to offer players more customization. Similar to the second example, Squish Run, I want to try and create color variations or maybe subtle changes in outfit in my characters, to create another element of interest and freedom for users to explore.

Design Strategies

As I am planning to incorporate options for the user to customize their character, I plan to choose color variations that are colorful but still maintain the more muted tones in the current design. I currently still like the personality that the more game-y/pixel type font VT323 offers the game design, so I plan to keep that in the current design system. I plan to match added iconography (e.g. for menu, pause, instructions, audio, and other icons) to this same pixel type style as well to create cohesion. I plan to leave the menu settings in the top right corner by the game title, as I want them to be clear and visible, but not a distracting part of the game or something that immediately attracts the player’s attention.

Interaction Strategies

To incorporate more user feedback, I plan to include subtle animations and audio on things like button clicks to provide the user with more visual and aural feedback. I also hope to include some ongoing animations within the game by using time-based behaviors to make the visuals a little less static. For example, having the penguins move subtly or playing with time-based behavior in audio tracks used. To incorporate the customization feature, I believe I will need to create another section that collects the user input for whichever color/outfit variation they choose for their character, and use JavaScript to assign/change the character designs accordingly.

Experience Goals

Through the design choices I’ve proposed, I hope to create a more enjoyable experience for my users. Nesting game rules and settings away not only clears up the visual design of the page, but also offers the user more freedom to navigate as they would like. By adding more subtle visual animations and audio cues, I hope to enhance the user experience by providing more feedback on user actions, as well as making the overall game more engaging. Additionally, I hope that the addition of character customization in allowing users to choose different character designs will offer another level of personalization that will make the game more fun and interesting.
